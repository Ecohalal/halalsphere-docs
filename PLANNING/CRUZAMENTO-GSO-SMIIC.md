## 2. MAPA DE CRUZAMENTO COMPLETO

### Legenda
- IDÊNTICO = Mesma letra, mesmo nome, mesma descrição
- EQUIVALENTE = Mesma letra, mesmo conceito mas texto diferente
- DIVERGENTE = Mesma letra, conteúdo diferente
- EXCLUSIVO = Existe em apenas uma norma

## 5. AS 4 DIVERGÊNCIAS CRÍTICAS

### 5.1 ABATE (a mais importante)

```
GSO:  C (Produção alimentícia) → CI (animal perecível) + CV (ABATE separado)
SMIIC: C (Produção alimentícia) → CI (ABATE + animal perecível) ← unificado
```

**Impacto**: Uma empresa de abate (frigorífico) será classificada como:
- No GSO: **CV** (Abate de animais)
- Na SMIIC: **CI** (Abate Halal e Processamento de produtos perecíveis)

**Para o sistema**: Precisa mapear CV ↔ CI entre normas, ou manter campo separado por norma.

### 5.2 COSMÉTICOS/TÊXTEIS/COURO

```
GSO:  K (Bioquímica) → inclui cosméticos, têxteis, couro como exemplos
SMIIC: K (Bioquímica) → SEM esses itens
       L (Outros) → LI (Cosméticos), LII (Têxteis), LIII (Couro)
```

**Impacto**: Uma empresa de cosméticos será:
- No GSO: **K** (Bioquímica)
- Na SMIIC: **LI** (Cosméticos dentro de "Outros materiais de processamento")

**Para o sistema**: Códigos diferentes, mapeamento obrigatório.

### 5.3 SERVIÇOS FINANCEIROS E TURISMO

```
GSO:  H (Serviços) → apenas segurança alimentar
SMIIC: H (Serviços) → HI (segurança) + HII (financeiros) + HIII (turismo)
```

**Impacto**: Certificação de bancos islâmicos e hotéis Muslim Friendly só é possível via SMIIC (HII/HIII). O GSO não cobre estes setores.

### 5.4 TRANSPORTE - SUBDIVISÃO GI

```
GSO GI:  Exclusivamente perecíveis
SMIIC GI: Perecíveis E estáveis à temperatura ambiente (escopo mais amplo)
```

**Impacto menor**: Na prática, a diferença é no escopo do que GI cobre.

## 8. RECOMENDAÇÃO PARA IMPLEMENTAÇÃO

### Opção Recomendada: Modelo Dual com Mapeamento

```
IndustrialGroup
  ├── code: "A"
  ├── name: "Criação de animais"
  ├── applicableStandards: ["GSO_2055_2", "SMIIC_02"]  ← NOVO
  └── categories[]
       ├── IndustrialCategory
       │    ├── code: "CV"
       │    ├── name: "Abate de animais"
       │    ├── gsoCode: "CV"          ← mapeamento GSO
       │    ├── smiicCode: "CI"        ← mapeamento SMIIC (compartilhado)
       │    ├── gsoAuditMode: "IN_LOCO"
       │    └── smiicAuditMode: "IN_LOCO"
       └── ...
```

### Campos a adicionar no Prisma Schema

```prisma
model IndustrialCategory {
  // ... campos existentes ...
  gsoCode        String?    // Código na norma GSO 2055-2
  smiicCode      String?    // Código na norma SMIIC 02
  gsoAuditMode   AuditMode? // PRESENCIAL ou REMOTO no GSO
  smiicAuditMode AuditMode? // PRESENCIAL ou REMOTO na SMIIC
  isGsoApplicable  Boolean @default(true)
  isSmiicApplicable Boolean @default(true)
}

enum AuditMode {
  IN_LOCO
  REMOTO
}
```

Isso permite:
1. Uma única tabela de categorias (sem duplicação)
2. Mapeamento correto para cada norma
3. Validação de auditoria presencial/remota por norma
4. Filtro de categorias por norma no wizard
5. Geração correta do código no certificado (GSO ou SMIIC)
