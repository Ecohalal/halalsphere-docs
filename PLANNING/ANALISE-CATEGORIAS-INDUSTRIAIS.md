### 3.5 Resumo das Divergências Críticas

| # | Divergência | Impacto |
|---|------------|---------|
| 1 | **Seed diz GSO 2055-2 mas não segue** | Certificados podem ser emitidos com categorias inválidas |
| 2 | **Display Map ≠ Seed ≠ Frontend** | Certificado mostra label diferente do que foi selecionado |
| 3 | **GSO tem 9 campos, Seed tem 11 grupos** | Categorias G-K do seed (Cosméticos, Farmacêuticos, etc.) não existem no GSO |
| 4 | **Frontend estático tem só 4 grupos** | Arquivo legado desatualizado, pode causar confusão |
| 5 | **Campo C no GSO = Processamento** | No seed, C = Abate. No display map, C = Abate de Aves |
| 6 | **Subcategorias do seed são inventadas** | BI-BVIII (8 tipos de processamento) não existem em nenhuma norma |
| 7 | **SMIIC 02 não está implementado** | Norma SMIIC tem estrutura com Grupos (a-e) acima das Categorias |
| 8 | **Ambas as normas coexistem no doc** | Precisa definir QUAL norma seguir, ou implementar ambas |

## 6. RECOMENDAÇÃO

Considerando que:
- O documento é da **FAMBRAS** (PR 7.1 Rev 24/04/2025)
- Contém **ambas as normas** (GSO 2055-2 e SMIIC 02)
- O sistema atual **não segue nenhuma** fielmente

**Recomendação**: Implementar **ambas as normas** (Opção C), pois:
1. O documento da FAMBRAS apresenta ambas como referência
2. Diferentes mercados/certificações podem exigir uma ou outra
3. Permite flexibilidade para o cliente escolher a norma aplicável
4. A estrutura do banco já suporta (3 níveis hierárquicos)

Para a Opção C, a implementação seria:
- Adicionar campo `standard` (enum: GSO_2055_2, SMIIC_02) ao modelo `IndustrialGroup`
- Seed com todos os dados de ambas as normas
- Wizard permite selecionar a norma primeiro, depois navegar nas categorias
